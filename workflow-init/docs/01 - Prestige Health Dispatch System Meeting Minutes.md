這是一場長達數小時的業務需求訪談紀錄，內容主要圍繞「院舍與護理員自動配對系統」的流程優化、系統整合、以及 ERP/配對平台（PO system）間的技術接口協作。以下是完整的整理：

***

## 🧭 會議簡要摘要（Summary）- 2025-11-17 AM

### 一、主要目標
- 建立一個「自動化護理工作配對系統」，以取代傳統人手 WhatsApp 配對方式。
- 確保系統能與現有 ERP 整合，資料雙向同步。
- 提升操作效率、減少人手錯誤、保持資料一致性和可追溯性。

***

### 二、系統核心流程（四大階段）

1. **員工開卡（Registration）**
   - 由現有 ERP 員工資料同步建立帳戶（包含姓名、電話、性別、證書等）。
   - 系統每日同步員工清單（例如早上 6 點刷新）。
   - 未持有效文件（如體檢、性罪條例、證書）者不得被派工。

2. **出 Post（Job Posting / 配對）**
   - 由內部人員或院舍建立工作需求（日期、時間、地點、服務內容等）。
   - 支援快速選擇機構（讀機構編號或輸入簡碼）。
   - 設置「優先名單（preference list）」：優先發送給曾服務過該院舍的護理員。

3. **報名與確認（Matching & Confirmation）**
   - 護理員透過 App 查看可接工單。
   - 兩種搶單機制：
     - **先到先得（first-come-first-serve）**
     - **評分制（scoring allocation）**：根據信譽分自動派單。
   - 護理員按下「預約／確認」→ 顯示註意事項（圖文）→ 確認後正式派工。
   - 系統自動發送 WhatsApp 通知。

4. **完成與結算（Completion & Settlement）**
   - 完成後系統自動檢查 ERP 出席紀錄，並更新狀態：
     - 出席 → 加分
     - 遲到／早退／請假 → 扣分
   - 48 小時內取消自動記錄「扣分事件」，人工覆核。
   - 完成紀錄存入「已完成列表」；支援出席報表或月結表輸出。

***

### 三、主要功能與業務規則

| 模組              | 功能說明                              |
| --------------- | --------------------------------- |
| 配對系統            | 自動 / 人手雙流程可切換（自動化優先）              |
| 員工資料同步          | 每日固定時段從 ERP 拉取更新                  |
| 院舍資料            | 支援 QRCODE/簡碼快速搜尋，含地區與地址           |
| 優先名單            | 每個院舍可保存特定熟悉人員清單                   |
| WhatsApp 整合     | 自動發放提醒、確認及註意事項（圖+文）               |
| 扣分機制            | 遲到、早退、請假或違規自動扣分（人工覆核接口保留）         |
| Optional：積分排名制  | 根據出勤分數決定接單優先級                     |
| Exception Cases | 特別情況可「人工 override」，人工確認後自動同步至 ERP |
| 安全控制            | 所有同步動作均不直接修改 ERP 原數據，以防資料錯亂       |

***

### 四、系統設計關鍵理念

- **用戶界面（UI/UX）**
  - 為護理員設計簡化、少文字、具圖像化的介面。
  - 一鍵操作、彈出式註意事項、預約確認流程明確。
- **自動化與人手彈性並存**
  - 預留人工覆核節點以應對院舍突發需求。
- **數據同步策略**
  - 採「被動式 API 拉取」+「主動式狀態更新」架構。
  - 每日固定時段（建議晚上 6 點至凌晨）同步資料以減少業務時段負擔。
- **抗錯性（Resilience）**
  - 系統需可處理網速慢、院舍改單、員工換電話等情況。
  - 任何異動須留 audit log 並可追蹤。

***

### 五、特別議題

1. **取消訂單流程**
   - 護理員 48 小時內取消 → 自動記錄／扣分 notificiation。
   - 需後臺人工確認並選擇替補人員。
   - 保留「扣錢 list」與「上班 list」分開。

2. **院舍臨時改單**
   - 需人工確認再發通知給護理員。
   - 不允許護理員自行在前端修改。

3. **上門服務（非院舍地址）**
   - 系統需支援另一欄「服務地址」欄位，供指定上門地點。
   - 若屬「上門服務」類別，地址必出現在派工單內。

4. **文件與有效期管理**
   - 體檢證、性罪條例、證書須設有效期。
   - 過期者自動標示「不可接工」。
