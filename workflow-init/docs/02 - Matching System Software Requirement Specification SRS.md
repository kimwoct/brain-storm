# 軟體需求規格文件 (Software Requirement Specification, SRS)  
## 院舍與護理員自動配對系統

***

## 1. 引言

### 1.1 目的  
本文件定義「院舍與護理員自動配對系統」之功能與非功能需求，作為系統設計、開發及驗收依據，並促進與現有ERP系統的整合協作。

### 1.2 範圍  
系統旨在替代現有人工作業配對流程，實現護理員配對之自動化及資料雙向同步，支援工作需求發布、護理員搶單、進度追蹤、評分與獎懲，並整合WhatsApp自動通知。

### 1.3 定義、縮寫與詞彙  
- ERP：企業資源規劃系統  
- 配對系統（Match System）：本計畫開發之護理員自動配對平台  
- 護理員（Helper）：接收派工的護理工作人員  
- 院舍（Organization）：護理服務提供場所  
- 預約搶單（Matching & Confirmation）：護理員從可接工單中搶單並確認接受  
- 加分/扣分：基於員工出席表現之評分機制  

***

## 2. 整體描述

### 2.1 主要目標  
- 開發自動化護理員配對系統，提高配對效率，減少人為錯誤  
- 與既有ERP系統雙向資料同步，確保信息準確且一致  
- 支援多場景操作，包括配置優先名單、搶單機制、例外狀況處理等  
- 採用易用的移動App介面與自動化通知（WhatsApp）  

### 2.2 利害關係人  
- 護理員  
- 院舍管理人員  
- 內部作業人員  
- IT/系統維運團隊  
- ERP系統  

### 2.3 假設與限制  
- ERP系統將提供必要資料接口和更新頻率  
- 系統不直接修改ERP主數據，所有更新需透過API或日志雙向同步  
- 護理員須通過有效文件認證，系統自動過濾無效者  

***

## 3. 功能需求

### 3.1 員工開卡與資料同步  
- ERP每日指定時間自動同步員工基本資料與證書狀態  
- 創建或更新護理員帳號，過期文件自動標示不可派工  
- 支援手動與自動雙重同步機制

### 3.2 工作需求發布（Post）  
- 內部或院舍可建立工作需求（日期、時間、地點、服務內容）  
- 支援QRCODE或簡碼快速搜尋院舍  
- 院舍可設定優先護理員名單，自動優先推送工單  

### 3.3 報名及確認配對機制  
- 護理員透過App瀏覽及搶工單  
- 搶單方式包含先到先得、評分制自動派單  
- 搶單後展示清楚註意事項與確認流程  
- 系統自動WhatsApp發送通知與提醒  

### 3.4 完成與結算管理  
- 自動對比ERP出席紀錄，依表現計算加／扣分  
- 48小時內取消事件自動記錄扣分，需後臺人工覆核  
- 支援出席及月結報表生成與匯出  

### 3.5 取消訂單及臨時改單處理  
- 48小時內取消自動扣分，後臺保留人工覆核與替補選項  
- 院舍臨時改單需人工核實並重新發通知，前端禁止自助修改  
- 上門服務地址需明確顯示在派工單上  

### 3.6 文件與有效期管理  
- 護理員證書、體檢、性罪條例文件設有效期管理  
- 過期自動標註並阻止派工  

### 3.7 安全及審計要求  
- 所有同步操作記錄審計日誌，可追溯操作歷史  
- 防止不當覆寫ERP主檔，採主-被動同步雙向機制  

***

## 4. 非功能需求

### 4.1 介面設計  
- 護理員App介面簡約，圖像化少文字，以提升操作便捷性  
- 操作流程一鍵完成，確保用戶體驗  

### 4.2 數據同步策略  
- 支援每日固定時間批次拉取（建議晚間18:00-次日凌晨）  
- 網路不穩時具備重試機制與異常處理  

### 4.3 性能與可靠性  
- 系統能處理大規模派單及資料同步  
- 支援異常退回與人工覆核確保資料正確  

***

## 5. 系統接口規範

### 5.1 員工資料同步 API  
- GET /api/employees?lastUpdated=timestamp  
- 返回員工詳細信息及證書狀態  

### 5.2 工作需求管理 API  
- POST /api/jobs/create  
- PATCH /api/jobs/{id}  
- GET /api/jobs?date=yyyy-mm-dd  

### 5.3 評分與考勤 API  
- PATCH /api/attendance/{id}  
- POST /api/penalty  

### 5.4 WhatsApp通知接口  
- POST /api/notifications/whatsapp  
- 參數包含接收者、模板、相關訂單ID等
